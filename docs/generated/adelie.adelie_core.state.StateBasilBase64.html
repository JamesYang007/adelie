abs_grad#

The \(\ell_2\) norms of grad across each group. abs_grad[i] is given by np.linalg.norm(grad[g:g+gs]) where g = groups[i] and gs = group_sizes[i].

active_sizes#

Active set size for every saved solution.

alpha#

Elastic net parameter. It must be in the range \([0,1]\).

benchmark_fit_active#

Fit time on the active set for a given BASIL iteration.

benchmark_fit_screen#

Fit time on the strong set for a given BASIL iteration.

benchmark_invariance#

Invariance time for a given BASIL iteration.

benchmark_kkt#

KKT time for a given BASIL iteration.

benchmark_screen#

Screen time for a given BASIL iteration.

betas#

betas[i] is the (untransformed) solution corresponding to lmdas[i].

early_exit#

True if the function should early exit based on training \(R^2\).

grad#

The true full gradient \(X_c^\top (y_c - X_c\beta)\) where \(\beta\) is given by screen_beta.

group_sizes#

List of group sizes corresponding to each element in groups. group_sizes[i] is the group size of the i th group.

groups#

List of starting indices to each group where G is the number of groups. groups[i] is the starting index of the i th group.

intercept#

True if the function should fit with intercept.

intercepts#

intercepts[i] is the intercept solution corresponding to lmdas[i].

lmda#

The regularization parameter at which the true solution is given by screen_beta.

lmda_max#

The smallest \(\lambda\) such that the true solution is zero for all coefficients that have a non-vanishing group lasso penalty (\(\ell_2\)-norm).

lmda_path#

The regularization path to solve for. The full path is not considered if early_exit is True. It is recommended that the path is sorted in decreasing order.

lmda_path_size#

Number of regularizations in the path if it is to be generated.

lmdas#

lmdas[i] is the i th \(\lambda\) regularization in the solution set.

max_iters#

Maximum number of coordinate descents.

max_screen_size#

Maximum number of strong groups allowed. The function will return a valid state and guaranteed to have strong set size less than or equal to max_screen_size.

min_ratio#

The ratio between the largest and smallest \(\lambda\) in the regularization sequence if it is to be generated.

n_threads#

Number of threads.

n_valid_solutions#

Number of valid solutions for a given BASIL iteration.

newton_max_iters#

Maximum number of iterations for the BCD update.

newton_tol#

Convergence tolerance for the BCD update.

penalty#

Penalty factor for each group in the same order as groups. It must be a non-negative vector.

pivot_slack_ratio#

If screening takes place, then pivot_slack_ratio number of groups with next smallest (new) active scores below the pivot point are also added to the strong set as slack. It is only used if screen_rule == "pivot".

pivot_subset_min#

If screening takes place, then at least pivot_subset_min number of gradient norms are used to determine the pivot point. It is only used if screen_rule == "pivot".

pivot_subset_ratio#

If screening takes place, then the (1 + pivot_subset_ratio) * s largest gradient norms are used to determine the pivot point where s is the current strong set size. It is only used if screen_rule == "pivot".

rsq#

The true unnormalized \(R^2\) given by \(\|y_c\|_2^2 - \|y_c-X_c\beta\|_2^2\) where \(\beta\) is given by screen_beta.

rsq_curv_tol#

Early stopping rule check on curvature of \(R^2\).

rsq_slope_tol#

Early stopping rule check on slope of \(R^2\).

rsq_tol#

Early stopping rule check on \(R^2\).

rsqs#

rsqs[i] is the (normalized) \(R^2\) at betas[i].

screen_begins#

List of indices that index a corresponding list of values for each strong group. screen_begins[i] is the starting index corresponding to the i th strong group. From this index, reading group_sizes[screen_set[i]] number of elements will grab values corresponding to the full i th strong group block.

screen_beta#

Coefficient vector on the strong set. screen_beta[b:b+p] is the coefficient for the i th strong group where k = screen_set[i], b = screen_begins[i], and p = group_sizes[k]. This must contain the true solution values for the strong groups.

screen_g1#

List of indices into screen_set that correspond to groups of size 1. screen_set[screen_g1[i]] is the i th strong group of size 1 such that group_sizes[screen_set[screen_g1[i]]] is 1.

screen_g2#

List of indices into screen_set that correspond to groups more than size 1. screen_set[screen_g2[i]] is the i th strong group of size more than 1 such that group_sizes[screen_set[screen_g2[i]]] is more than 1.

screen_hashset#

Hashmap containing the same values as screen_set. It is used to check if a given group is strong or not.

screen_is_active#

Boolean vector that indicates whether each strong group in groups is active or not. screen_is_active[i] is True if and only if screen_set[i] is active.

screen_order#

Ordering such that groups is sorted in ascending order for the strong groups. groups[screen_set[i]] is the i th strong group in ascending order.

screen_rule#

Strong rule type.

screen_set#

List of indices into groups that correspond to the strong groups. screen_set[i] is i th strong group. screen_set must contain at least the true (optimal) active groups when the regularization is given by lmda.

screen_sizes#

Strong set size for every saved solution.

setup_lmda_max#

True if the function should setup \(\lambda_\max\).

setup_lmda_path#

True if the function should setup the regularization path.

tol#

Convergence tolerance.
